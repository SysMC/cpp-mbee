Проект предназначен для поддержки программного обеспечения беспроводных радиомодулей MBee, разрабатываемых и производимых фирмой "Системы, модули и компоненты".

Версия компилятора для Windows, использованного при разработке библиотеки: g++ (i686-posix-dwarf-rev0, Built by MinGW-W64 project) 8.1.0.
Версия компилятора для Linux (Raspbian Stretch), использованного при разработке библиотеки: g++ (Raspbian 6.3.0-18+rpi1+deb9u1) 6.3.0 20170516.
Компилятор для микроконтроллеров MSP430 IAR C/C++ Compiler for MSP430 7.12.1 (7.12.1.987).
Компилятор для микроконтроллеров MSP430 TI v18.12.2.LTS (Code Composer Studio 9.1.0.00010).
Компилятор для микроконтроллеров STM32 IAR C/C++ Compiler for Arm version 8.40.2.
Компилятор для микроконтроллеров STM32 IDE SW4STM32 ARM Compiler for MCU 1.17.0.201812190825.


Поддерживаемое ПО:
1. SeralStar (http://sysmc.ru/documentation/sw_SerialStar.pdf).

Документация на библиотеку находится в cpp-mbee\docs\html\index.html.
Примеры располагаются в cpp-mbee\examples в каталогах, соответствующих целевым платформам. Внутри этих папок могут находиться отдельные readme-файлы, содержащие дополнительную информацию. Как правило, комментариев, содержащихся в исходных текстах примеров бывает
достаточно для понимания принципов работы с библиотекой.

Для ускорения изучения библиотеки и начала создания собственных проектов, рекомендуется в начале работы протестировать имеющиеся примеры. Впоследствии ими можно пользоваться как готовыми шаблонами при создании собственных приложений.

14.07.2018
1. Первая опубликованная стабильная версия библиотеки.

09.06.2019
1. Файлы SerialStar.h, MBee.h, SerialStar.cpp и MBee.cpp перекодированы с помощью Notepad++ из UTF-8 BOM в UTF-8. Перекодировка потребовалась чтобы компилятор IAR не выдавал ошибку "Unrecognized token".
2. В файле SerialStar.cpp  в определении метода uint32_t RxIoSampleResponse::getCounter(uint8_t number) добавлено явное преобразование типов при сдвиге. Преобразование потребовалось, чтобы IAR не генерировал "Warning: shift count is too
large".

17.06.2019 
1. Для исключения rollover`а счетчика миллисекунд, прошедших после включения питания, через 49 дней, тип возвращаемого значения функции millis() изменен с uint32_t на  uint64_t.
2. Тип переменных, которым присваивается значение millis() во всех функциях изменен с unsigned long на unsigned long long.

17.07.2019
1. В классRxIoSampleResponse добавлен метод getSampleNumber(uint8_t pin). Метод предназначен для упрощения процедуры разбора пакета 0x83.

19.07.2019
1. В файл SerialStar.h добавлено определение константы PWM_DUTY_CYCLE_MAX.

06.09.2019
1. Значение константы MAX_FRAME_DATA_SIZE изменено с 45 на 44 в соответствии с максимально возможной длиной фрейма (фрейм 0x0F).
2. В методе readPacket() исправлена ошибка при проверке максимальной длины пакета. К константе MAX_FRAME_DATA_SIZE добавлено 4 (длина служебных полей перед полем данных). Ошибка приводила к невозможности приема пакетов 0x81 и 0x8F с длиной поля данных более
37 байт.

12.09.2019
1. Тип возвращаемого значения функции millis() изменен с uint64_t на  uint32_t. Изменение вызвано ограниченными ресурсами микроконтроллеров MSP430 серии "Value line", а также отсутствием реальной необходимости обеспечивать в пользовательском приложении задержки
соизмеримые с 49 днями. Опасность rollover`а парируется требованием использовать при вычислении задержек только беззнаковые целочисленные переменные, а также не пользоваться конструкциями типа "end = start + x", вместо которых следует применять 
"x - start < или > end", где x - это текущее значение millis().
